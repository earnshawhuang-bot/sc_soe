3
Join & Enrich 数据关联
以 SO 号为主键，LEFT JOIN 所有数据源
so_master_df — SO 级别全量关联表
sc_df LEFT JOIN shipped_df ON SO号 → 已发货量 & 发货日期
sc_df LEFT JOIN fg_df ON SO号 → 在库量
sc_df LEFT JOIN pp_sched_df / pp_unsched_df ON SO号 → 排产信息 & 计划完工日期
sc_df LEFT JOIN gr_enriched_df ON SO号 → 实际入库量
sc_df LEFT JOIN lp_ks_df / lp_idn_df / lp_idn_dom_df ON SO号 → 装货日期 (Required Loading Date)
⚠️ GR → SO 间接关联：gr_df 没有 SO 号，必须先通过 gr.订单号 → pp.工单号 → pp.SO 中转得到 gr_enriched_df，再参与 JOIN

4
Status Engine 状态判定 + Gap 计算
对每张 SO 判定交付状态，计算生产与装运的时间差

状态判定逻辑（按优先级从高到低）

①✅
Fully Shipped
shipped_qty ≥ order_qty → 已全部发货
②🟡
Partially Shipped
shipped_qty > 0 但 < order_qty → 部分发货
③🟢
In Stock
fg_qty > 0 → 已产出在库，待发货
④🟢
Produced (GR)
gr_qty > 0 → 已生产入库确认
⑤⏳
In Production
有 planned_end_date → 排产中，需算 Gap
⑥🟠
Planned Only
有工单但无排期 → 高风险
⑦🔴
No Plan
SC 中有但 PP 中完全找不到 → 极高风险

Gap 计算（仅对 ⑤ In Production）

Gap = Loading Date − (Planned End Date + 1天)

+1天：生产完工后次日视为最早可入库/装货时间

🔴
Gap < 0 — 生产滞后 |Gap| 天
需要催产 或 推后船期
🟡
Gap = 0 ~ 2 — 紧张，几乎无缓冲
需要密切关注
🟢
Gap > 2 — 有缓冲
正常推进

风险等级汇总

🔴🔴 Critical
No Plan — 有订单但完全无生产安排
🔴 High
Gap < 0 或 Planned Only — 生产滞后 / 未排具体日期
🟡 Medium
Gap 0~2 或 Partial Shipped — 紧张但尚可调度
🟢 Low
Gap > 2 / In Stock / Fully Shipped — 正常
